# Screenshots for App Store Submission

You need to create the following screenshots for PWABuilder and app store submissions.

## Required Screenshots

### Mobile (Narrow) - 750x1334 or similar portrait ratio
- `home-narrow.png` - Home dashboard view
- `shopping-narrow.png` - Shopping lists view
- `tasks-narrow.png` - Tasks/Todos view
- `recipes-narrow.png` - Recipes view

### Desktop/Tablet (Wide) - 1280x720 or similar landscape ratio
- `home-wide.png` - Home dashboard view

## How to Create Screenshots

### Option 1: Browser DevTools
1. Open your app in Chrome
2. Press F12 to open DevTools
3. Click the device toolbar icon (or Ctrl+Shift+M)
4. Select iPhone 14 Pro (390x844) or similar
5. Take a screenshot (Ctrl+Shift+P â†’ "Capture screenshot")
6. Resize to 750x1334 if needed

### Option 2: Use a Screenshot Tool
- [Shottr](https://shottr.cc/) (Mac)
- [Greenshot](https://getgreenshot.org/) (Windows)
- [Flameshot](https://flameshot.org/) (Linux)

## App Store Requirements

### Google Play Store
- Minimum 2 screenshots
- JPEG or PNG, max 8MB each
- Sizes: Phone (16:9 ratio), Tablet (16:9), Wear OS (1:1)

### Apple App Store
- 6.7" (iPhone 14 Pro Max): 1290 x 2796
- 6.5" (iPhone 11 Pro Max): 1284 x 2778
- 5.5" (iPhone 8 Plus): 1242 x 2208
- iPad Pro 12.9": 2048 x 2732

### Microsoft Store
- At least 1 screenshot
- PNG, 1366x768 minimum

## Tips
- Show the app in its best light
- Use real data (or realistic mock data)
- Highlight key features
- Keep it clean and professional
